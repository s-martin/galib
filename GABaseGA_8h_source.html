86 /* ----------------------------------------------------------------------------
87  The base GA class is virtual - it defines the core data elements and parts
88 of the interface that are common to all genetic algorithms (as defined in
89 GAlib, that is).
90 
91 initialize
92  Undefined for the base class. The initialization routine typically calls
93 the population initializer (which typically calls the genome initializers).
94 It should also reset the statistics.
95 
96 step
97  Evolve by one generation. 'generation' can be defined different ways for
98 different genetic algorithms, but in the traditional formulation a generation
99 mean creation of a new population (or portion thereof).
100 
101 done
102  Calls the completion measure routine to tell whether or not the GA is done.
103 
104 evolve
105  This method is provided as a convenience so that you don't have to increment
106 the GA generation-by-generation by hand. If you do decide to do it by hand,
107 be sure that you initialize before you start evolving!
108 ---------------------------------------------------------------------------- */
