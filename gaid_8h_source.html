81 // The ID numbers for built-in classes are enumerated in the GAID class. Sorry
82 // but I had to do the dimension at the end of the names rather than at the
83 // beginning since you cannot use a digit to start a variable name.
84 // If you derive your own polymorphic class or specialize a
85 // template class, then give it its own ID number and class name in the
86 // specialization (see the string and real number specializations for examples)
87 // Use these macros to define the identity for your class. If you do not use
88 // these macros then your object will be identity-less.
89 // Here's how to use the macro.
90 //
91 // class GASimpleGA : public GA {
92 // public:
93 // GADefineClassIdentity("GASimpleGA", 1);
94 // ...
95 // };
96 //
97 // Notice that your template classes will all have the same name using this
98 // method unless you specialize. For example, GA1DArrayGenome<char> and
99 // GA1DArrayGenome<int> will both have the name GA1DArrayGenome as well as
100 // the same number. Beware.
101 
102 #define GADefineIdentity(name, id) \
103  const char *className() const override { return name; } \
104  int classID() const override { return id; }
