18 /* ----------------------------------------------------------------------------
19 1DBinaryStringGenome
20 -------------------------------------------------------------------------------
21 resize
22  These genomes are resizable. In addition, you can set a resize behaviour
23  to specify the bounds in which the resize can occur.
24 
25 copy
26  Copy bits from the specified genome using the location and length
27  that are passed to us. If the current genome is too short for the
28  entire length, copy whatever we can. If the original genome is too
29  short for the specified length, copy whatever we can. If either location
30  is out of bounds, return without doing anything.
31  We do NOT check for negative values in the locations!
32  This routine clips if the copy sizes do not match - it does NOT resize the
33  genome to fit the copy. You'll have to do resizes before you call this
34  routine if you want the copy to fit the original.
35 
36 ==, !=
37  Are two genomes equal? Our test for equality is based upon the
38  contents of the genome, NOT the behaviour. So as long as the bitstreams
39  match, the genomes are 'equal'. This means that a resizeable genome
40  may be equal to a fixed-length genome. But a chromsome with 500
41  bits allocated is not equal to a genome with 10 bits allocated unless
42  both are the same size.
43 ---------------------------------------------------------------------------- */
