16 /* ----------------------------------------------------------------------------
17  GAList
18 -------------------------------------------------------------------------------
19  This object is a container for nodes that have a list structure. The base
20 list object is responsible for maintaining the list heirarchy. This object is
21 responsible for doing the memory management (allocating and de-allocating the
22 nodes in the list). We insulate the user entirely from nodes - when you use
23 a list, you don't get nodes back, you get the contents of nodes (ie the user
24 doesn't have to think about the list parts of a node, they can simply assume
25 that their data is organized into a list structure).
26  We include an iterator in this object so that you can navigate through the
27 list. You can create another iterator and assign it to your list so you can
28 have multiple iterators.
29  All of the actions take place relative to the current location of the
30 embedded iterator. None of the iterators change the state of the list. Be
31 careful so that you don't end up with an iterator dangling with a pointer to
32 a part of a list that no longer exists (I would need some kind of reference
33 counting and/or message passing to take care of this at a lower level, and I'm
34 not ready to implement that at this point).
35  For now we allocate nodes on the fly. Eventually I would like to do some
36 better memory management (arrays perhaps?) so we don't have to do so much
37 alloc and dealloc and recursion.
38  We depend on the template-ized GAListIter routine, thus the declaration.
39 
40 copy
41  Make a complete copy of the list and return a pointer to the new list.
42 
43 remove
44  Remove the current node from the list, free the memory used by the node, and
45  return a pointer to a copy of the object that the node contained.
46 
47 destroy
48  Remove the current node from the list and free the memory it was using.
49  Destroys the object that the node contained.
50 
51 insert - list
52  Inserts the contents of list in to the current list and removes it from the
53  original list. Does NOT delete the original list, but DOES assume
54  responsibility for the memory used by the contents of the original list.
55 
56 insert - object
57  Insert the object into the list at the specified place relative to the
58  current location of the embedded iterator.
59 
60 current, head, tail, next, prev, warp
61  These iterator methods are defined as access to the embedded iterator of the
62  list. Use these methods to move the insertion point and to traverse the
63  list. You can also create other iterators for this list, but they won't
64  affect the contents.
65 ---------------------------------------------------------------------------- */

71 // Copy a node, including all of its siblings up to the end of the list. We do
72 // NOT wrap around the list until we return the first node - we stop at the
73 // tail of the list. Return the pointer to the first node in the list.
74 template <class T> GANode<T> *_GAListCopy(GANode<T> *node, GANode<T> *head)
75 {
76  if (!node)
77  return nullptr;
78  auto *newnode = new GANode<T>(node->contents);
79  GANode<T> *lasttmp = newnode, *newtmp = nullptr;
80  GANode<T> *tmp = DYN_CAST(GANode<T> *, node->next);
81  while (tmp && tmp != head)
82  {
83  newtmp = new GANode<T>(tmp->contents);
84  newtmp->prev = lasttmp;
85  lasttmp->next = newtmp;
86 
87  lasttmp = newtmp;
88  tmp = DYN_CAST(GANode<T> *, tmp->next);
89  }
90  if (newtmp)
91  {
92  newtmp->next = newnode;
93  newnode->prev = newtmp;
94  }
95  else
96  {
97  newnode->next = newnode;
98  newnode->prev = newnode;
99  }
100  return newnode;
101 }

195  // Swap two nodes in the list. This has no effect on the size or the
196  // iterator. If either index is out of bounds then we don't do anything.
197  int swap(unsigned int a, unsigned int b)
198  {
199  if (a == b || a > (unsigned int)size() || b > (unsigned int)size())
200  return GAListBASE::NO_ERR;
201  GANodeBASE *tmp = hd, *anode = nullptr, *bnode = nullptr;
202  unsigned int cur = 0;
203  while (tmp && tmp->next != hd)
204  {
205  if (a == cur)
206  anode = tmp;
207  if (b == cur)
208  bnode = tmp;
209  tmp = tmp->next;
210  cur++;
211  }
212  if (a == cur)
213  anode = tmp;
214  if (b == cur)

301 /* ----------------------------------------------------------------------------
302  GAListIter
303 -------------------------------------------------------------------------------
304  This is a type-safe derivation of the base ListIter object. I copied the
305 methods from the base class (I know, a no-no) rather than doing calls to the
306 base class methods.
307  We depend on the template-ized GAList, thus the declaration.
308  Behaviour for the iterator methods is defined as follows. If the current
309 node is null, attempts to access a derived position from the current position
310 will return NULL. The only way to reset the current node is to call the head()
311 locater (you always have to start at the list head to navigate the list). If
312 the current node is non-null and the derived node is null, the current node is
313 NOT changed, but NULL is returned. You can also warp to a new position if you
314 have another iterator or a list with an embedded iterator.
315  When we create a new list iterator, it defaults to the same node as the one
316 used to create it. If it is created with a list as its argument, it defaults
317 to the list's iterator's current position.
318 ---------------------------------------------------------------------------- */
