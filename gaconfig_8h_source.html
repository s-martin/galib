15 /* ----------------------------------------------------------------------------
16 PREPROCESSOR DIRECTIVES
17 
18  Here are the preprocessor directives that the library understands.
19 
20  Some of these are already set up for the OSes with which I am familiar. See
21 below for the pre-defined sets. If you come up with a compiler/platform
22 configuration that is not listed here, please send it to me so that i can
23 incorporate it into the code base.
24 
25  GALIB_USE_PID
26 
27  Define this if the system has a getpid function that
28  returns something sane and useful.
29 
30  GALIB_USE_BORLAND_INST
31 
32  For compilers that use the Borland instantiation model.
33  These compilers expect all of the template code to be
34  in the header file. The Cfront model, on the other
35  hand, expects source files with names similar to the
36  header files, but all of the template code does not
37  need to be in the header files.
38 
39  When you define this flag, the source file that
40  corresponds to the header file is explicitly included
41  at the end of the header file for all headers that
42  contain templates.
43 
44  GALIB_USE_AUTO_INST
45 
46  For compilers that do not do automatic instantiation
47  (such as g++ version 2.6.8) you will have to force
48  instantiations. When this flag is not defined, GAlib
49  forces an instantiation of all of the template classes
50  that it uses (such as real genome and string genome).
51 
52 
53  GALIB_USE_RAN1 These specify which random number function to use. Only
54  GALIB_USE_RAN2 one of these may be specified. You may have to tweak
55  GALIB_USE_RAN3 random.h a bit as well (these functions are not defined
56  GALIB_USE_RAND the same way on each platform). For best results, use
57  GALIB_USE_RANDOM ran2 or ran3 (performance is slightly slower than most
58  GALIB_USE_RAND48 system RNGs, but you'll get better results).
59 
60  If you want to use another random number generator you
61  must hack random.h directly (see the comments in that
62  file).
63 
64  GALIB_BITBASE The built-in type to use for bit conversions. This
65  should be set to the type of the largest integer that
66  your system supports. If you have long long int then
67  use it. If you don't plan to use more than 16 or 32
68  bits to represent your binary-to-decimal mappings then
69  you can use something smaller (long int for example).
70  If you do not set this, GAlib will automatically use
71  the size of a long int. The bitbase determines the
72  maximum number of bits you can use to represent a
73  decimal number in the binary-to-decimal genomes.
74 
75  GALIB_BITS_IN_WORD How many bits are in a word? For many systems, a word is
76  a char and is 8 bits long.
77 
78 ---------------------------------------------------------------------------- */

146 // you want a better RNG, you can use the others in GAlib, but you'll have to
147 // do a bit of checking to make sure it works with your DOS/Windows config.
148 // All of the RNGs work fine under all of the 32-bit OSes I've tried, but they
